## Opt 1 (LOW risk): connect() re-runs idempotent DDL every call

Fix (behavior-preserving): make schema/FTS setup run-once. Keep a module-level set of already-initialized `db_path` strings; skip `executescript`+`_ensure_fts` when the path is known-initialized in THIS process. Gate the 2× `COUNT(*)` FTS backfill behind a cheap guard so it only runs on the one-time empty-FTS-but-nonempty-memories migration condition (e.g. `SELECT 1 FROM memories LIMIT 1` cheap precheck, or only when FTS was just created).
CRITICAL: a brand-new `db_path` (fresh process / new file) MUST still get the full schema + FTS on first connect — correctness preserved. Only repeated connects to an already-initialized path in the same process skip the redundant DDL.
Invariant: first connect to any db_path behaves identically (full schema/FTS); subsequent same-process connects are just faster. ~0.41ms saved per LLM call.
